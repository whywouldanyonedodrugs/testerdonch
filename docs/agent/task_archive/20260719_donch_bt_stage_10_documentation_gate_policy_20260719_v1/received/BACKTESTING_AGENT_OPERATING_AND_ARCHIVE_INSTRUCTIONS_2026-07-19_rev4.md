# Backtesting Agent Operating and Archive Instructions

## Start

Verify root, `AGENTS.md`, branch/commit/remotes, working tree, commands, data roots, source manifests, archive convention and Drive target. Use an isolated worktree for consequential changes. Preserve unrelated user files.

## Boundaries

```text
rankable_interval: [2023-01-01, 2026-01-01)
protected_period: 2026 onward
paid_vendor_data: prohibited
orders_private_account_actions: prohibited
economic_run: exact human approval only
Capitalcom_real_payload: exact intake/read approval only
commit_push_merge: exact task approval only
```

Existing identities, roots, hashes and terminal decisions are immutable.

## Gate routing

Future research must separate:

```text
integrity
economic_relevance
generality_context
controls
independent_validation
deployment
```

Do not encode one universal `all_gates_pass` as the only research route.

Every economic contract records:

```text
payoff_archetype
intended_claim_scope
hard_gates
routing_diagnostics
control_eligibility
validation_requirements
```

Past contracts retain their original gates.

A positive definition that fails only temporal, regime, symbol or market-day concentration may be archived as a conditional or narrow-sleeve candidate without being promoted. Preserve the failed unconditional decision.

Controls may be authorized for a positive conditional object when the exact task permits them. Controls remain unexecuted without approval.

## Post-hoc context

When outcomes motivate a context hypothesis:

- label it post-hoc;
- do not use calendar year as the context;
- use only causal pre-existing variables;
- register all context branches;
- keep same-sample results explanatory;
- require independent or prospective evidence.

## Evidence package

Compact handoffs include:

- binding artifact manifest;
- exact contracts and definition registers;
- gate and route matrix;
- binding contributor identities for time, symbol, episode and market day;
- all attempted, killed and omitted branches;
- review of why each failure is integrity, economics, uncertainty, context, execution or sample limitation.

## Completion

Report terminal machine decision and research route separately. A route does not overwrite the run decision or evidence level.
