# Research Efficiency and Search Discipline

## Objective

Use computation to distinguish:

- no edge;
- conditional edge;
- convex edge;
- execution-sensitive edge;
- sample-limited edge;
- invalid evidence.

Do not use compute to polish one historical winner.

## Funnel

```text
verified immutable data
-> causal reusable features
-> frozen mechanism and payoff archetype
-> frozen candidate/control identities
-> mechanical and cost kill checks
-> exact simulation
-> route classification
-> mechanism controls and ablations
-> duplicate/episode/context clustering
-> purged or nested validation
-> independent/prospective evidence
-> portfolio and deployment review
```

Early stages may reject or route. They cannot validate.

## Efficient conditional research

When a candidate is positive but concentrated:

1. preserve the unconditional run and decision;
2. identify the binding concentration identity;
3. register a post-hoc context hypothesis;
4. restrict candidate contexts to pre-existing causal variables;
5. run all registered context branches and matched out-of-context controls;
6. cluster by market episode and context episode;
7. require independent evidence before promotion.

Do not use `year`, `best symbol`, or `best month` as the context.

## Safe search

- cached causal features keyed by data and feature hashes;
- deterministic candidate, control and cluster identities;
- few mechanism-relevant alternatives rather than grids;
- predeclared payoff archetypes;
- one primary and one robustness translation where possible;
- all attempts retained;
- exact execution only for mechanically valid candidates;
- contributor identities in compact evidence.

## Dangerous shortcuts

- selecting contexts from return rankings;
- changing payoff archetype after seeing median/skew;
- treating concentration as proof of either failure or edge without context analysis;
- relaxing costs because gross returns are close;
- combining correlated weak sleeves into a portfolio before controls;
- using protected data to define a context;
- rerunning a closed translation under cosmetic parameter changes.

## Stop and route rules

Close an exact translation when:

- base economics are absent under its intended implementation;
- clean controls falsify the distinct mechanism;
- execution uncertainty dominates and cannot be resolved;
- further variants are cosmetic.

Preserve a separate route when:

- positive base economics fail only generality/context gates;
- a predeclared convex distribution has recurring tails;
- the opportunity is coherent but sample-limited;
- a materially different observable mechanism becomes available.
