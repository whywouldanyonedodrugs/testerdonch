# Research Efficiency and Search Discipline, rev4

## Efficient funnel

```text
shared causal feature cache
-> outcome-free magnitude/coverage audit
-> batched registered response surfaces
-> sparse candidate beam
-> next-block evaluation
-> rolling replication
-> controls and prospective evidence
```

Do not spend a full confirmatory run on a guessed threshold when a low-cost development surface can establish sign, shape, magnitude and horizon first.

## Compute strategy

- scan shared data once for several related hypotheses;
- cache fold-safe features by data/feature/fold hash;
- vectorize bins, horizons and ablations;
- use bounded-memory columnar processing;
- checkpoint campaigns transactionally;
- retain all attempts without retaining redundant raw tables;
- continue independent family lanes after family-specific failure.

## Search discipline

Explore enough to characterize the mechanism, but constrain complexity:

- predeclared features and transformations;
- limited interaction depth;
- sparse models and rule lists;
- maximum candidate beam;
- deterministic tie-breaks;
- nested evaluation;
- family/program multiplicity.

Search breadth is not overfitting when it is isolated to development folds and honestly counted. Hidden search or selection on evaluation folds is overfitting.
