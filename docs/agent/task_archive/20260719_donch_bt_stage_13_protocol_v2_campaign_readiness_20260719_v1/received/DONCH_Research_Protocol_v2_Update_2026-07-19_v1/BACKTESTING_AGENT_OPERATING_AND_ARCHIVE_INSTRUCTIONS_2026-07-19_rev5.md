# Backtesting Agent Operating and Archive Instructions, rev5

Verify repository, authority, data, protected boundary and campaign manifest before work.

For protocol-v2 tasks:

- distinguish family from translation;
- preserve all old terminal decisions;
- permit registered redevelopment with new IDs and inherited multiplicity;
- never hide searched cells;
- use fold-safe cached features;
- enforce no backward information transfer;
- report program exposure class;
- follow campaign-level phase permissions exactly.

For unattended campaigns:

- use transactional campaign state and idempotent restart;
- continue unaffected family lanes after family-specific negative results;
- stop globally on shared authority, protected, timestamp, Git or replay failure;
- produce heartbeats and resource checks;
- obtain independent reviews required by the manifest;
- reconcile every planned, executed, skipped and failed cell.

A campaign-level human approval can authorize several named economic runs only when the exact manifest, search spaces, folds, selection algorithm, costs, phase permissions and stop rules are attached and hashed.
