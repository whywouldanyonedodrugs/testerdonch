## Reusable component conclusions

### Completed failure

The clearest shared component is a completed failure after a defined parent state and countertrend move. It should be specified as a causal state transition, not reconstructed by combining the best historical thresholds from separate families.

### Entry structure

Mandatory retest waiting is not supported by the current breakout comparison. A future continuation test may model:

```text
immediate starter
+ optional, separately accounted retest add
```

That requires exact multi-lot accounting, which the current common infrastructure does not provide.

### Continuous context

Prior-high distance, relative strength, BTC impulse, breadth, session, and funding state should default to context or continuous ranks. They become hard gates only after a predeclared component test shows stable incremental value.

### Mechanism over pattern

The next hypothesis must name who is compelled to trade, why the pressure should persist or fail, what Kraken-observable state distinguishes the mechanism, and what null control would falsify it. Another bar pattern is not a mechanism.

July 2026 may calibrate fills, spread, slippage, staleness, and book reconstruction for these contracts. It may not choose their economic rules.

- funding-prediction revision shocks;
- OI-vacuum continuation or reversal;
- assignment, covered-liquidation, or unwind aftermath;
- order-book withdrawal and forced-flow sequencing;
- maker-style liquidity provision;
- short-horizon funding-boundary inventory effects.

One uneven protected week is a capture pilot, not a parameter-selection sample.
