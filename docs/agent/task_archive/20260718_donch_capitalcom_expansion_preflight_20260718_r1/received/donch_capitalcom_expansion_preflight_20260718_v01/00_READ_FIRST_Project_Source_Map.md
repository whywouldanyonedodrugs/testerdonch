Use this bundle for analysis and planning. It does not authorize a backtest, an economic screen, inspection of protected strategy outcomes, capture restart, an order, a risk change, repository mutation, project-source replacement, or deletion of originals.

- execution venue: Kraken only;
- rankable historical research: 2023-01-01 through 2025-12-31;

## Authority order

1. Machine-enforced contracts and finalized run manifests.
2. Authoritative run roots, hashes, ledgers, and audit artifacts.
3. Current master continuity brief.
4. Current active manuals and data-capability reports.
5. Current strategic audits and agent reports.
6. Research reports and hypothesis catalogues as priors.
7. Superseded plans, old venue manuals, and legacy Donch material as provenance.

If sources conflict, record the conflict and use the higher authority. A narrative summary does not override a machine contract.

## Five separate states

Do not collapse these states into one label:

- **Hypothesis presence:** an idea or mechanism has been recorded.
- **Evidence:** observations or event-level artifacts support a bounded claim.
- **Reproducibility:** code, data, configuration, universe, funding, and run identity can be reconstructed.
- **Validation:** a frozen candidate has passed the approved train-only and later holdout protocol.
- **Deployment:** an independently approved live operating decision exists.

No current family is validation-grade or live-ready. A clean evidence package is not deployment authorization.

## Compact versus local-only material

The core and optional bundles contain Markdown and CSV only. Original PDFs remain offline. Their page-bounded text extractions are local synthesis inputs, not upload-ready canonical sources. Large external-review bundles and row-level payloads also remain local; their compact decision-bearing summaries live under `01_review_package`.

The full original file inventory, hashes, duplicate map, and authority map live under `00_audit`. No original was intentionally replaced or deleted by this curation pass.

## When a new task arrives

Before acting, state:

1. the requested decision or artifact;
2. the environment in scope;
3. the data period and venue;
4. whether any protected data could be touched;
5. the highest authority that governs the change;
6. the acceptance criteria and verification;
7. the human approvals required.

If the prompt is not self-contained for a remote repository, stop and request the missing files, paths, contracts, or expected tests. Do not assume that the Donch ChatGPT project, backtesting repository, and capture repository share files.
