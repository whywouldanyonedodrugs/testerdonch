## 6. A2 parent and context design

A2 is an overlay, not an independent trade generator.

Implement one exact grammar. Prefer a staged conditional-template model:

1. A1 and A3 parent candidates are developed without hidden A2 promotion.
2. Parent beam slots are frozen deterministically.
3. Pre-registered A2 templates bind to exact family/fold/beam slots.
4. Parent-only counterpart and context-overlay identity are generated
   atomically before A2 outcome access.
5. Missing parent slot means `unavailable_no_parent`, never reassignment.
6. Paired event IDs, exposure, gross return, costs, funding and occupancy are
   identical except for the context permission/size action.
7. Context promotion requires a registered paired-uplift rule and control.
8. Legacy A2 rows with inconsistent `parent_family`/`parent_binding` remain
   lineage defects unless an unambiguous source correction is proven.

The agent may choose a simpler exact grammar if it provides complete typed
identities and preserves the mechanism. Record the choice before outcomes.

## 9. Efficient economic campaign architecture

Design the future approved run for minimum wasted computation without losing
important information.

### 9.1 Shared semantic caches

Build and reuse hash-bound caches for:

```text
decision calendar
PIT universe and lagged liquidity
daily and 5-minute feature panels
prior highs/lows
ATR and volatility estimators
breadth/dispersion
BTC/ETH context
funding
entry/exit lookup arrays
event keys
```

A harmless wrapper/report change must not invalidate a semantic cache. A change
to feature meaning, source manifest, schema, protected boundary or selected
event keys must invalidate it.

### 9.2 Aggregate-first coarse evaluation

Evaluate every final registered configuration using exact aggregate summaries.
Do not write full event ledgers for all definitions.

Persist enough statistics to reproduce selection:

```text
event count
market days
symbol/month/year contributions
base and stress net
median/trimmed mean
inner-fold vector including empties
coverage
concentration
opportunity frequency
component metrics
```

### 9.3 Conditional materialization

Materialize full event ledgers only for a predeclared set:

```text
all deterministic beam survivors
all mechanism anchors and main component nulls
near-misses under a frozen rule
a stratified random failed audit sample
objects needed for controls/forensics
```

The near-miss and failed-audit rules must be frozen before outcomes.

### 9.4 Safe pruning

Safe pre-outcome pruning:

```text
invalid combinations
duplicate economic addresses
inactive-axis duplicates
unavailable required data
failed source/hash/integrity gate
impossible minimum sample or coverage after complete data enumeration
```

Safe stage-boundary pruning after complete registered development:

```text
no eligible plateau/candidate under the frozen rule
component-null equality showing exact economic duplication
family-specific integrity failure
```

Do not cancel a configuration merely because its first chronological folds,
symbols or months are unprofitable. That can discard regime-dependent
opportunities and biases the research.

Do not use stochastic bandit/successive-halving pruning unless it is separately
registered and proves that all information required for the final response
surface is retained. Default: do not use it.

Outer evaluation and controls are already the main compute-saving gates:

- only frozen beam candidates enter outer folds;
- only frozen survivors receive conditional controls;
- unrelated families continue after a family stop.

A family may stop future economic work when the frozen route makes success
logically impossible and the remaining folds cannot affect any permitted
scientific claim. If later-fold behavior would still inform regime/context
interpretation, continue the fixed diagnostic subset instead of declaring the
family exhausted.

### 9.5 Runtime

Benchmark and choose the maximum independently safe worker count not exceeding:

```text
logical CPU count
memory ceiling
storage write throughput
```

Use four workers on the currently reported four-logical-CPU host unless a fresh
benchmark proves a better safe setting.

Use:

```text
persistent detached supervisor
lazy bounded scheduling
atomic state
idempotent restart
30-minute Telegram heartbeats
worker-death retry
no-progress detection
renewable checkpoints
no interactive polling
```

Memory and free-disk reserves are safety boundaries, not economic rejection
rules. Compact and hash-archive closed artifacts before storage pressure. Never
delete unique or unverified evidence.

Incrementally publish hash-verified artifacts when a measurement, family,
development fold, beam, outer fold, control tranche or forensic stage completes.
Do not expose partial winner rankings through Telegram and do not use interim
outcomes to change the run.
