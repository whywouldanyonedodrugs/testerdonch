
### Vectorized aggregate-first execution

- batch addresses sharing family/fold/feature signatures;
- vectorize across configurations where semantics permit;
- use columnar/memory-mapped data;
- avoid repeated parsing and full-frame copies;
- use one bounded artifact writer or atomic per-worker partitions to prevent I/O
  contention;
- compute exact aggregate summaries for every executable address;
- do not write a full event ledger for every failed address.

### Conditional expensive work

Perform full materialization, controls, bootstraps and forensics only for the
frozen materialization set and survivors defined above.

### Safe pruning

Pre-outcome/compile-time pruning is permitted for:

```text
invalid combinations
duplicate economic addresses
inactive-axis duplicates
unavailable required data
failed authority/integrity gates
mathematically impossible minimum observation/coverage requirements after the
    complete eligible population is enumerated
```

Stage-boundary pruning is permitted only after the complete registered
development surface relevant to that gate exists:

```text
no eligible plateau/candidate under the frozen rule
exact component-null economic duplication
family-specific integrity failure
no frozen parent for conditional controls
```

Do not cancel an address because early folds, symbols, months, or partial PnL are
poor. Do not implement return-based successive halving or a bandit scheduler.
That would remove regime and convexity information and introduce a new searched
stopping rule.

Order work to maximize cache reuse, not to inspect likely winners first.

Record every pruned attempt with an exact deterministic reason. Pruning reduces
compute; it does not reduce multiplicity or erase negative information.
